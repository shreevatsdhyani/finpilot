# Security Policy

## Reporting a vulnerability

Please do not open public issues for security vulnerabilities.

Report suspected vulnerabilities privately to the maintainers with:

1. A clear description of the issue
2. Reproduction steps and affected files/endpoints
3. Potential impact and suggested mitigation

We will acknowledge reports as quickly as possible and provide updates while
the issue is being triaged and resolved.

## Scope

This policy covers the full monorepo, including:

1. `backend/` (FastAPI services and API routes)
2. `frontend/` (Next.js application)
3. Infrastructure and configuration files at repository root
